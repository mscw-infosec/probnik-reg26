# Privesc 9

## Описание

Вам нужно получить root доступ к системе и прочитать флаг. 
Доступ к системе есть под пользователем bob (пароль bob123).

## Решение

Сначала нужно подключиться к целевой машине по SSH, используя предоставленные учетные данные.

Пользователь: `bob`
Пароль: `bob123`


```bash
ssh bob@<ip_адрес>
```

Теперь нужно найти способ повысить свои привилегии. Посмотрим на файл `/etc/passwd`. Это один из первых файлов, который стоит проверять при попадании в систему.

Файл `/etc/passwd` содержит информацию о пользователях системы. В старых системах в этом файле также хранились хэши паролей. В современных системах для этого используется файл `/etc/shadow`, который недоступен для чтения обычным пользователям.

Проверим содержимое `/etc/passwd`:

```bash
cat /etc/passwd
```

Вы увидите строки, описывающие каждого пользователя. Нас интересует строка для `root`:

```
root:some_password_here:0:0:root:/root:/bin/bash
```

В данном задании, из-за неправильной конфигурации сервера, вместо `x` (указывающего на то, что пароль в `/etc/shadow`) или хэша, в этом файле находится пароль пользователя root в открытом виде.

Теперь, когда у нас есть пароль от `root`, мы можем переключиться на этого пользователя. Для этого используется команда `su`.

```bash
su -
```

Система запросит пароль. После получения доступа для рута, можно прочитать флаг.

```bash
cat /root/flag.txt
```

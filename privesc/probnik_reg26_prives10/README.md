# Privesc 10

## Описание
В этой задаче нужно получить root доступ к системе. Пользователь для входа - bob (пароль: bob123).

```
ssh -p <port> bob@<domain>
```

## Решение

Это задание на повышение привилегий состоит из двух этапов: сначала горизонтальное перемещение (от пользователя `bob` к `eve`), а затем вертикальное (от `eve` к `root`).


Начинаем с подключения к серверу по SSH с известными учетными данными.

Пользователь: `bob`
Пароль: `bob123`
```bash
ssh bob@<domain> -p <port>
```

1. Осмотримся в домашней директории: `ls -la`
2. В папке `.ssh` находим приватный SSH-ключ для пользователя `eve`. Скорее всего, он по ошибке оказался доступен для чтения пользователю `bob`. Скопируем его содержимое себе на локальную машину.

Теперь, имея приватный SSH-ключ `eve`, мы можем войти в систему от ее имени.

Подключитесь как `eve`: Используйте опцию `-i` для указания пути к файлу с ключом.
```bash
ssh eve@<domain> -p <port> -i eve_key
```
Теперь вы в системе как пользователь `eve`.

Остался финальный шаг - получить права суперпользователя.

1. Проверим права `sudo`: Узнаем, какие команды `eve` может выполнять с помощью `sudo`:
    ```bash
    sudo -l
    ```
2. Мы увидим, что `eve` может выполнять команду `su` от имени `root` без пароля:
    ```
    (root) NOPASSWD: /usr/bin/su
    ```
3. Получим `root` шелл:
    ```bash
    sudo su -
    ```

Теперь, когда мы `root`, прочитаем файл с флагом:
```bash
cat /root/flag.txt
```
